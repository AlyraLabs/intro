How it works

Alyra provides end-to-end execution â€” from intent parsing and route validation to deterministic on-chain settlement.

[ 01 ]

Intent

Application

OFF-CHAIN / USER SIDE

A user triggers an action (swap, bridge, transfer) through an app using the Alyra API/SDK.The app forms an Intent with all transaction parameters and sends it to the Alyra Endpoint.

Alyra Endpoint

OFF-CHAIN / CORE

The Endpoint receives the Intent and queries connected DEXs, bridges, and solvers to collect quotes for route construction.

[ 02 ]

Validate

Alyra Backend

OFF-CHAIN / CORE

Alyra Backend collects quotes from connected DEXs, bridges, and solvers, builds possible routes, and selects the most efficient one. The chosen route is sent to the security layer for verification.

Security Stack

OFF-CHAIN / VERIFICATION

The security layer simulates the route off-chain, checks call integrity, and scans for threat patterns or anomalies. After successful validation, the execution packet is returned to the backend and then returned to the application for user signing.

[ 03 ]

Sign

USER

OFF-CHAIN / USER SIDE

After receiving the validated route, the application displays the final execution quote to the user. The user reviews all parameters and signs it with their wallet, generating a verified execution packet ready for on-chain submission.

[ 04 ]

Execute

Alyra Diamond Contract

ON-CHAIN / EXECUTION

The signed execution packet is submitted to the Alyra Diamond, built on the EIP-2535 Diamond Standard (multi-facet architecture). The contract validates the packet and executes the route through the proper Facet (Bridge or DEX). Settlement finalizes on the destination chain exactly as defined.
